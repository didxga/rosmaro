Leaves are basic building blocks which are bags of functions. Every leaf may cause a transition. It's also possible to add special functions which are executed when a leaf is entered or left (see below). Functions of leaves may return values which may be Promises. Every leaf has access to the context of the whole model. It can also read its own unique id.

Here's an example of a leaf which uses all of its features:

const rich_leaf = {

  unsynchronized: ["follow_your_path"],

  on_entry() {
    console.log(`Entering the rich leaf.`)
  },

  follow_your_path() {
    console.log(this.get_introduction())
    const new_context = { ...this.context, new_value: 42 }
    this.follow("my_path", new_context)
  },

  before_leave() {
    console.log(`Leaving the rich leaf.`)
  },

  after_leave() {
    console.log(`The rich leaf has been left.`)
  }

  get_introduction() {
    return `My id is ${this.id}.`
  }

}

The lack of the type property indicates that this is a leaf.

Special methods and properties

on_entry

Called when a transition to this leaf occurs. It has access to the id and the context. It may also cause a transition.

before_leave

Called before the leaf is left. It occurs before the on_entry method of the new leaf is called. It has access to the id and the context, but it cannot cause a transition.

after_leave

Called after the leaf is left. It occurs after the on_entry method of the new leaf is called. It has access to the id and the context, but it cannot cause a transition.

unsynchronized

A list of methods that don't need to be synchronized using the locking mechanism. It may be ignored in case of composed leaves. For more details check the chapter on locking.

Unique ID

Every method of a leaf may read this.id which is a unique ID of that leaf. It changes only when a transition from this leaf occurs. It changes even when it's a loop, that is a transition going from that leaf to the same leaf. It happens because even though the leaf may stay the same, a transition may set some different context, and thus the behavior of this leaf may differ. That's why a new ID is assigned.

const id_reader = {
  read_id() {
    return this.id
  }
}

Context

The context is an immutable and serializable object shared among all leaves. It is available as the this.context property. At the beginning, before any transition occurs, it's an empty object. It may be set during a transition.

const context_reader = {
  read_context() {
    return this.context
  }
}

const context_setter = {
  set_new_context() {
    this.follow("some_arrow", new_context)
  }
}
