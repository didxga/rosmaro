  • Adapters

An adapter is a kind of a wrapper for another node. It provides methods to translate the context for the adapted node as well as rename all the arrows that leave the adapted node.

Let's think of a situation when we want to re-use some node, but it doesn't exactly fit our model. It expects the context to look like this:

{ number: 42 }

While the rest of our model works with the following format:

{ value: 42 }

We also expect that the node is going to follow an arrow named found, while the incompatible node actually follows an arrow named discovered. Situations like that are very common. The good thing is that adapters address this problem:

const adapted = {
  type: "adapter",
  map_entering_context(ctx) {
    return { number: ctx.value }
  },
  map_leaving_context(ctx) {
    return { value: ctx.number }
  },
  rename_leaving_arrows: {
    discovered: "found"
  },
  adapted: incompatible
}

A very important part is the type property which in case of adapters must equal adapter.
