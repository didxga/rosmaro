Interface

The storage mechanism has the following interface:

{
  async get_data() {},
  async set_data(new_state) {},
  async remove_data() {}
}

get_data()

Returns a Promise that resolves to an object equal to the one previously set using the set_data method. If nothing has been set yet or the remove_data method has been called, it returns a nullify value.

set_data(new_state)

Takes a serializable object representing the state of a Rosmaro model and returns a Promise which when resolved indicates the data has been saved.

remove_data

Returns a Promise which when resolved indicates all the data of that model has been removed.

Errors

Every method of a storage may throw an error. It is important that those errors are Error objects. Throwing something else, like a string, may cause a crash.
