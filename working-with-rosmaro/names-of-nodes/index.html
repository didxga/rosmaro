  • Names of nodes

This chapter covers both types of names of Rosmaro nodes.

Short names used in the context of a single node

Leaves

Leaves don't specify any names.

Nodes of a graph

const graph = {
  type: "graph",
  start: "A",
  arrows: {},
  nodes: { A: any_node_definition, B: any_node_definition }
}

In the above example there are two nodes called A and B.

Nodes of a composite

const composite = {
  type: "composite",
  nodes: [
    ["A", any_node_definition],
    ["B", any_node_definition]
  ]
}

This composite consist of two nodes: A and B. Bear in mind how are they defined. It's not an object, but an array. Also order of elements matters - the name goes first, then the definition.

Adapted node

const adapter = {
  type: "adapter",
  adapted: any_node_definition
}

The adapted node is always called adapted.

Full names used by the whole model

Full names of nodes are names given them by their parents together with full names of those parents glued with the : symbol.

Nested graphs


//named "C" by its parent "B" what gives "B:C"
const C = {}

//named "B" by its parent
const B = {
  type: "graph",
  start: "C",
  arrows: {},
  nodes: { C }
}

//"" (root)
const A = {
  type: "graph",
  start: "B",
  arrows: {},
  nodes: { B }
}

Here we can see one leaf C. Its parent is B and the parent of B is A. But A has no parent so it has no name. That's why the full name of C is B:C.

Named composites

If we put a composite with two leaves into any node that may name it, like another composite, graph or adapter, we get two full names with a common parent:

const D = {}

const C = {}

const B = {
  type: "composite",
  nodes: [
    ["C", C],
    ["D", D]
  ]
}

//A is the root
const A = {
  type: "adapter",
  adapted: B
}

The leaves are: adapted:C, adapted:D.
