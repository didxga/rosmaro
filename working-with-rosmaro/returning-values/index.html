  • Returning values

Let's assume that we have a Rosmaro model built with the following leaf.

const the_one_who_knows = {
  give_me_the_answer() {
    return 42
  }
}

The Rosmaro model will always return a Promise, even if the called method doesn't return a Promise. That's why we always need to wait for the result.

const the_result = await model.give_me_the_answer()

The awaited result is always an object with keys named after full names of nodes. In the above case it could look like this:

{
  "they:the_one_who_knows": 42,
  "they:the_one_who_sees_nothing": undefined
}

There are many returned values when composite nodes are used to build the model. Regardless the number of current nodes, the result always follows that format. So even if there would be just one active node, it would still look like this:

{
  "they:the_one_who_knows": 42
}
